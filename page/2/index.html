console.log() 라는 친구에 대해서 설명을 해야겠네요. node.js 세상에는 console 이라는 친구가 있어요. 님 이름이 뭔지 모르겠지만 님도 이름이 있잖아요. 쟤는 이름이 console이에요. 근데, console이라는 애는 log를 남길 수 있어요. log는 통나무예요. 가 아니라. 기록이라고 볼 수 있어요. 프로그램을 만들면 곳곳에 로그를 남겨요. 어디까지 실행됐는지 남기거나 어떤 값을 들춰볼 때도 쓰고요. 혹은 에러가 나도 저기다가 남겨요.

디버깅의 기본이라고 부를 수 있죠. 로그를 잘 배워놔야 잘 쓸 수 있어요. 그리고 뭐가 잘못됐는지 알 수 있을 거예요.

console.log() 라고 하면 console 이라는 친구야 너가 가지고 있는 것 중에 log라는 걸 꺼내. 그리고 실행시켜 이런 뜻이에요.
console.log 에서 이렇게 .으로 연결되면 console이 가지고 있는 것 중 어떤 걸 부르는 거예요. 다른 예로 console.error도 있어요. 그러니까 console이라는 애가 log도 갖고 있고 error도 갖고 있는데 걔가 가지고 있는 애 중 하나를 가져오는 거예요.

2. 개발을 하기 위한 환경 준비

* node.js

먼저 node.js를 설치합니다. node.js 공식 홈페이지

버전이 두 개 존재합니다.

  • 8.10.0 LTS
  • 9.8.0 현재버전

버전은 계속 업그레이드되기 때문에 다를 수 있습니다. 어떤 버전이든 상관이 없습니다.

설치하는 과정은 간단합니다. 다운로드 받은 파일을 실행하고 계속 다음을 눌러주세요.

설치가 완료된 정상적으로 완료되었는지 확인하는 과정이 반드시 필요합니다. 터미널에서 다음 명령어를 입력하세요.

  • 터미널이 뭔지 모르시는 분에게
    명령 프롬프트 또는 터미널 또는 커맨드 라인 인터페이스(CLI)라고 부르는 프로그램입니다. 윈도우를 쓰신다면 왼쪽 하단의 윈도우 버튼을 누르신 후 “명령 프롬프트”라고 입력하시면 찾으실 수 있습니다.

윈도우에서 단축키로 명령 프롬프트를 여는 방법은 다음과 같습니다.

  • 윈도우 버튼 + r 를 입력, “실행” 프로그램을 띄웁니다.
  • 실행 프로그램에서 열기: 부분에 cmd를 입력하고 확인을 누릅니다.

참고로 윈도우 키는 일반적으로 키보드 좌측 Ctrl 키와 Alt 키 사이에 있어요.

맥의 경우엔 Spotlight에서 터미널이라고 입력하시면 찾을 수 있습니다.

* Visual Studio Code

Visual Studio Code 공식 홈페이지에서 다운받아 설치하십시오.

Visual Studio Code(이하, vscode)는 개발할 때 사용하는 텍스트 에디터입니다. 이와 비슷한 프로그램으로 Sublime Text, Atom 등이 있습니다. 필자는 javascript 개발 시 vscode를 사용하기 때문에 이 글에서도 이 프로그램으로 진행할 것입니다.

위의 Download for Windows 버튼으로 다운로드 받으십시오. 타 OS를 사용하시는 경우 해당 버전에 맞는 다운로드 버튼이 나타납니다.

설치 과정이 필요하시면 이 글을 참고하셔도 좋을 것 같습니다.

한 가지 Tip을 드리자면 설치 도중 “Code(으)로 열기”작업을 Windows 탐색기 파일의 상황에 맞는 메뉴에 추가, 디렉터리의 상황에 맞는 메뉴에 추가 항목을 체크하세요.

위의 사진처럼 폴더나 파일을 마우스 우클릭 시 Open with Code가 생깁니다. 파일이나 폴더를 vscode로 열기 수월해집니다.

* ngrok

ngrok 홈페이지서 다운로드 받으세요. 가입도 필요합니다.

다운 받으시면 아시겠지만 ngrok라는 프로그램은 단일 실행파일로 이루어져 있습니다. 이 프로그램은 더블클릭으로 실행할 수 있는 프로그램이 아니며, 터미널을 통해 명령어로 동작하게 됩니다.
이 프로그램은 사용자의 서버를 https 서버로 만들어줄 수 있습니다.

https 서버란

HTTPS(Hypertext Transfer Protocol over Secure Socket Layer, HTTP over TLS, HTTP over SSL, HTTP Secure)는 월드 와이드 웹 통신 프로토콜인 HTTP의 보안이 강화된 버전이다.

[위키백과] [https://ko.wikipedia.org/wiki/HTTPS] [HTTPS]

기본적으로 https 서버를 구동하기 위해서는 도메인과 인증서가 필요합니다. 그러나 위 프로그램을 사용할 경우 이 둘을 단순하게 해결해줍니다. 그 방법은 차후 설명하겠습니다.
